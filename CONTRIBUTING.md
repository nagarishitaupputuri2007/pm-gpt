---

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your change:
   ```bash
   git checkout -b your-branch-name
   ```

---

## Development Guidelines

- Keep changes **focused and minimal**
- Follow existing code style and structure
- Write clear, descriptive commit messages  
  (e.g. `feat: add framework comparison logic`, `docs: update README`)
- Avoid unrelated changes in the same commit

---

## Testing

If your contribution affects logic or data flow:
- Ensure the app runs without errors
- Add or update tests where applicable

---

## Submitting Changes

1. Push your branch to your fork
2. Open a Pull Request to the `main` branch
3. Clearly describe:
   - What you changed
   - Why the change is needed

Pull requests will be reviewed as soon as possible.

---

## Reporting Issues

If you find a bug or have a suggestion:
- Open a GitHub Issue
- Provide clear steps to reproduce (if applicable)
- Include screenshots or logs when helpful
